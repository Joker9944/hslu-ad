## Observations

1. Bubble Sort random is ~5× slower than reversed (7514 vs 1557ms at 100k) — this is actually correct. It's the classic "turtle problem": large elements bubble right quickly, but small elements at the end can only move left one step per pass. A random array scatters small elements everywhere, making it much worse than reversed.
2. Selection Sort sorted (629ms) is ~2× faster than reversed (1681ms) despite both being O(n²) algorithmically — Selection Sort does the same number of comparisons regardless of input, but the sorted array barely ever triggers the "new minimum found" branch, making it very branch-predictor-friendly. Reversed input triggers it on almost every comparison.
3. Selection Sort reversed (1681ms) > random (1230ms) — same reason: reversed input causes maximum branch mispredictions in the inner-loop minimum search.
